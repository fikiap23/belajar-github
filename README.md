# belajar-github

Belajar github dari web programming unpas

Berikut ini adalah tabel dengan perintah-perintah Git lengkap beserta contoh penggunaan dan penjelasannya:

| Perintah Git   | Contoh                                                   | Penjelasan                                                                                                                                                                                                                                                                                                     |
| -------------- | -------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `git init`     | `git init my-repo`                                       | Membuat repositori Git baru di direktori saat ini atau dengan nama tertentu. Ini akan membuat direktori `.git` yang berisi semua metadata dan objek yang diperlukan oleh Git.                                                                                                                                  |
| `git clone`    | `git clone https://github.com/user/repo.git`             | Mengkloning repositori Git yang ada ke dalam direktori lokal. Ini akan membuat salinan lengkap dari repositori, termasuk semua riwayat perubahan (commits), cabang (branches), dan tag (tags).                                                                                                                 |
| `git add`      | `git add file.txt`                                       | Menambahkan perubahan pada file ke dalam daftar perubahan Git. Ini menandai file tertentu untuk disertakan dalam commit berikutnya. Jika ingin menambahkan semua perubahan pada file-file yang ada, dapat menggunakan `git add .`.                                                                             |
| `git commit`   | `git commit -m "Menambahkan fitur baru"`                 | Membuat snapshot (snapshot) perubahan yang sudah ditambahkan dengan pesan komit. Ini menyimpan perubahan yang ditambahkan ke repositori secara permanen. Pesan komit digunakan untuk memberikan deskripsi singkat tentang perubahan yang dilakukan.                                                            |
| `git push`     | `git push origin main`                                   | Mengirimkan perubahan lokal ke repositori jarak jauh (remote repository). Perintah ini mengunggah perubahan yang sudah di-commit ke repositori pusat yang terkait dengan cabang tertentu. `origin` adalah nama default untuk repositori pusat. `main` adalah nama cabang yang akan diunggah.                   |
| `git pull`     | `git pull origin main`                                   | Mengambil perubahan terbaru dari repositori jarak jauh dan menggabungkannya dengan perubahan lokal. Perintah ini mengunduh perubahan yang ada di repositori pusat dan melakukan penggabungan (merge) otomatis dengan cabang yang sedang aktif di repositori lokal.                                             |
| `git branch`   | `git branch new-feature`                                 | Membuat cabang baru dalam repositori untuk pengembangan paralel. Perintah ini digunakan untuk membuat cabang baru di repositori Git. `new-feature` adalah nama cabang baru yang akan dibuat.                                                                                                                   |
| `git checkout` | `git checkout new-feature`                               | Beralih ke cabang atau komit tertentu dalam repositori. Perintah ini digunakan untuk beralih antara cabang-cabang yang ada dalam repositori atau berpindah ke komit tertentu. `new-feature` adalah nama cabang atau referensi komit yang akan dituju.                                                          |
| `git merge`    | `git merge new-feature`                                  | Menggabungkan perubahan dari cabang tertentu ke cabang saat ini. Perintah ini digunakan untuk menggabungkan perubahan dari satu cabang ke cabang aktif saat ini. `new-feature` adalah nama cabang yang akan digabungkan.                                                                                       |
| `git status`   | `git status`                                             | Menampilkan status perubahan saat ini dalam repositori. Perintah ini memberikan informasi tentang status file-file yang ada di repositori saat ini, seperti file-file yang sudah diubah tetapi belum ditambahkan atau file-file yang sudah ditambahkan tetapi belum di-commit.                                 |
| `git log`      | `git log --oneline`                                      | Menampilkan riwayat komit dalam format yang ringkas. Perintah ini menampilkan riwayat komit yang ada di repositori, termasuk informasi seperti ID komit, penulis, waktu, dan pesan komit. `--oneline` digunakan untuk tampilan yang lebih kompak.                                                              |
| `git remote`   | `git remote add origin https://github.com/user/repo.git` | Mengatur repositori jarak jauh yang dihubungkan dengan repositori lokal. Perintah ini digunakan untuk menambahkan repositori jarak jauh yang akan digunakan sebagai repositori pusat untuk mengirim dan menerima perubahan. `origin` adalah nama yang diberikan kepada repositori jarak jauh tersebut.         |
| `git stash`    | `git stash save "Perubahan sementara"`                   | Menyembunyikan perubahan sementara yang belum siap untuk dikomit. Perintah ini digunakan untuk menyimpan perubahan sementara yang belum di-commit ke dalam penyimpanan sementara. Hal ini berguna ketika Anda ingin beralih ke cabang atau komit lain tanpa mengkomit perubahan yang belum selesai.            |
| `git reset`    | `git reset HEAD file.txt`                                | Membatalkan penambahan file ke daftar perubahan sebelum di-commit. Perintah ini digunakan untuk menghapus file-file dari daftar perubahan setelah menggunakan `git add`, sehingga file-file tersebut tidak akan dimasukkan ke dalam commit berikutnya. `HEAD` merujuk ke komit terakhir dalam cabang saat ini. |

Perintah-perintah di atas merupakan beberapa perintah Git yang umum digunakan dalam pengembangan perangkat lunak dengan Git. Harap diingat bahwa contoh dan penjelasan yang diberikan di atas hanya representasi umum, dan penggunaan sebenarnya mungkin membutuhkan opsi tambahan atau parameter khusus. Pastikan untuk membaca dokumentasi resmi Git atau menggunakan `git --help` untuk mempelajari lebih lanjut tentang setiap perintah.
